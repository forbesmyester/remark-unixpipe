remark-unixpipe will, for a Code Block with the lang “unixpipe”, pipe it’s body through the command specified in it’s meta.

This means the following code:

This is above

```unixpipe env PUNCTUATION="!" ./res/shell_runner | sed 's/^/    /'
Hello ${NAME}${PUNCTUATION}
```

This is below

Results in:

This is above

$ INPUT="$(cat /dev/stdin)"
$ echo "$INPUT"
Hello ${NAME}${PUNCTUATION}
$ OUTPUT="$(echo "$INPUT" | env NAME="World" envsubst)"
$ echo "$OUTPUT"
Hello World!

This is below

When the contents of ./res/shell_runner is

#!/bin/bash

IFS=$'\n\t'
set -euo pipefail

trap 'echo "\$ $BASH_COMMAND"' DEBUG

INPUT="$(cat /dev/stdin)"
echo "$INPUT"
OUTPUT="$(echo "$INPUT" | env NAME="World" envsubst)"
echo "$OUTPUT"
